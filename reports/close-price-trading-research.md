# 收盤價交易策略研究

## 背景

我們發現台股市值前百存在系統性「開高走低」偏差（日均 -0.089%），導致 151 週回測期間：

| 價格基準 | 累積市場收益 |
|----------|-------------|
| Open-to-Open | +3.46% |
| Close-to-Close | +28.72% |

差距高達 **25 個百分點**。

同時，qlib 的 Label 定義為 close-to-close 收益：

```python
label_expr = "Ref($close, -2) / Ref($close, -1) - 1"
```

但我們使用 open 價格交易（`trade_price="open"`），產生訓練目標與實際執行的 mismatch。

---

## 1. 為何 Qlib 使用收盤價

學術界與 qlib 框架選擇收盤價作為標準交易價格，有四個核心原因：

### 1.1 資訊完整性

收盤價反映了一整天的市場資訊，是當日所有資訊充分定價後的均衡結果。開盤價則只反映了隔夜累積的資訊，尚未經過盤中交易驗證。

Lou, Polk & Skouras (2019) 的研究指出，隔夜收益（close-to-open）與日內收益（open-to-close）呈現截然不同的因子結構——動量效應集中在隔夜，反轉效應集中在日內。這表明開盤價和收盤價捕捉的是不同的資訊集合，而收盤價的資訊集合更完整。

### 1.2 微觀結構噪音

開盤集合競價機制本身就產生額外噪音：

- **台股 8:30-9:00 撮合**：訂單積累 30 分鐘後一次撮合，價格發現效率較低
- **隔夜資訊衝擊**：重大新聞、國際市場變動在開盤瞬間反映，容易過度反應
- **流動性不均**：開盤前的買賣單分佈不對稱，導致價格偏離真實價值

相比之下，收盤價經過一整天的連續交易驗證，是微觀結構噪音最低的價格點。

### 1.3 機構行為分化

機構投資者的行為在日內呈現明顯分化：

- **隔夜動量**（close-to-open）：反映機構的隔夜風險偏好和資訊優勢
- **日內反轉**（open-to-close）：反映散戶追漲殺跌後的均值回歸

這種分化意味著：用開盤價訓練的模型可能學到的是「隔夜動量」信號，而非我們真正想捕捉的「日內價值發現」信號。收盤價作為終點，更能代表市場對股票價值的共識。

### 1.4 模型穩定性

Fama-French 三因子、五因子等經典模型都使用收盤價計算收益率。原因：

- Close-to-close 序列更平滑、更可重現
- 開盤價的噪音會降低因子 IC，增加模型不穩定性
- 學術文獻的可比性要求統一基準

### Qlib 社群討論

qlib GitHub Issue #1514 中有開發者實測：

> 將 Label 從 close-to-close 改為 open-to-open 後，IC 從 0.04 提升到 0.07。

但 qlib 團隊仍選擇收盤價作為預設，理由是收盤價是「更安全的選擇」——IC 的提升可能來自開盤價噪音的過擬合，而非真實的預測能力提升。

## 2. 台股日內偏差數據（實證分析）

基於我們的 qlib 資料庫（市值前 100、363 個交易日），分析結果：

### 2.1 系統性日內收益

| 指標 | 數值 |
|------|------|
| 日均盤中收益 (Open→Close) | **-0.089%** |
| 年化影響 | 約 -22% |
| 正收益天數佔比 | ~46% |

### 2.2 星期效應

| 星期 | 日均盤中收益 |
|------|-------------|
| Mon | **-0.246%** |
| Tue | **-0.198%** |
| Wed | -0.027% |
| Thu | -0.032% |
| Fri | +0.056% |

週一、週二的「開高走低」最嚴重，週五則接近持平甚至微正。

### 2.3 價格位置分析

以日內 High-Low 範圍為基準（Low=0, High=1）：

| 價格 | 日內位置 | 含義 |
|------|---------|------|
| Open | 0.512 | 略高於中位 |
| Close | 0.465 | 略低於中位 |
| VWAP (估算) | ~0.49 | 接近中位 |

### 2.4 與 VWAP 的偏差

| 比較 | 偏差 |
|------|------|
| Open vs VWAP | +0.035%（Open 偏高） |
| Close vs VWAP | -0.075%（Close 偏低） |

**結論**：台股市值前百存在穩定的「開高走低」模式。Open 系統性偏高、Close 系統性偏低，用 Open 買入相當於每天多付 ~0.089% 的隱性成本。

## 3. 價格何時收斂至收盤價

### 3.1 台股波動率結構：L 型

台股（9:00-13:30，僅 4.5 小時）的日內波動率呈 **L 型**：

```
波動率
  ↑
  █████
  █    ██
  █      ████
  █          █████████████
  █                       ████
  └─────────────────────────────→ 時間
  9:00  10:00  11:00  12:00  13:30
```

- 開盤後波動率最高（資訊衝擊 + 流動性不足）
- 上午逐步下降
- **午後 12:00 起趨於穩定**
- 13:25-13:30 收盤集合競價有小幅上升

這與美股的 **U 型**（開盤高、午盤低、收盤高）不同，台股因交易時間短，不存在「尾盤波動率再升」的現象。

### 3.2 價格收斂時間估計

基於波動率結構和國際市場研究：

| 時段 | 距收盤 | 價格偏離收盤估計 | 依據 |
|------|--------|-----------------|------|
| 9:00-10:00 | 3.5-4.5h | ±0.5-1.0% | 開盤衝擊期 |
| 10:00-11:00 | 2.5-3.5h | ±0.3-0.5% | 資訊消化期 |
| 11:00-12:00 | 1.5-2.5h | ±0.2-0.3% | 趨於穩定 |
| **12:00-12:30** | **1-1.5h** | **±0.15-0.2%** | **開始收斂** |
| **12:30-13:20** | **10-55min** | **±0.1-0.15%** | **建議執行窗口** |
| 13:25-13:30 | 0-5min | ≈0% | 收盤集合競價 |

美股的參考數據：S&P 500 有 65.6% 的時間收盤價在 14:00（收盤前 2 小時）價格的 ±0.20% 內。考慮台股交易時段更短、L 型波動率更陡，收斂時間可能更早。

### 3.3 建議執行窗口：12:30-13:20

選擇此窗口的理由：

1. **波動率已低**：午後波動率接近全日最低
2. **流動性充足**：連續競價仍在進行，掛單深度合理
3. **時間充裕**：50 分鐘的窗口足以分批執行
4. **避開收盤集合競價**：13:25-13:30 的集合競價可能有大單造成價格壓力

## 4. 執行策略建議

### 方案比較

| 方案 | 價格偏離 | 流動性 | 實施難度 | 適用情境 |
|------|---------|--------|---------|---------|
| **A. MOC 收盤委託** (13:25-13:30) | ≈0% | 集中但可能滑價 | 低 | 小資金、追求精確 |
| **B. 收盤前執行** (12:30-13:20) | ±0.1-0.15% | 好 | 中 | 中等資金、需要執行品質 |
| **C. 改 Label 為 open-to-open** | N/A（對齊） | N/A | 高（需重新訓練） | 堅持用 open 交易 |

### 方案 A：MOC 收盤委託

```
優點：價格完全等於收盤價，訓練與執行完美對齊
缺點：
  - 台股收盤集合競價僅 5 分鐘（13:25-13:30）
  - 所有 MOC 訂單集中在此窗口，大單容易滑價
  - 無法預知最終撮合價格
```

### 方案 B：收盤前分批執行（推薦）

```
優點：
  - 50 分鐘的執行窗口，可分批掛單
  - 流動性好，滑價可控
  - 價格已接近收盤（偏離 ±0.1-0.15%）
缺點：
  - 非零偏差，但遠小於 open 的 0.089%/天
```

### 方案 C：改 Label 為 open-to-open

```
優點：訓練目標與交易執行完全對齊
缺點：
  - 需重新訓練所有模型
  - Open 噪音大，模型 IC 可能下降
  - 違背學術共識，可比性降低
```

### 建議

**短期（立即可行）**：採用方案 B，將交易執行時間從開盤改到 12:30-13:20。

- 預期效果：每日減少 ~0.089% 的隱性成本（open 偏差），年化節省 ~22%
- 回測驗證：將 `trade_price` 改為 `"close"` 重跑回測，觀察策略在 close 基準下的表現

**中期（需要數據支持）**：取得盤中分鐘線，精確量化各時段價格與收盤價的偏差，找到最佳執行時點。

## 5. 數據限制

### 目前可用資料

| 資料 | 頻率 | 來源 |
|------|------|------|
| OHLCV | 日頻 | TWSE / FinMind / yfinance |
| 籌碼面 | 日頻 | TWSE / FinMind |

### 無法精確驗證的部分

1. **盤中分鐘線**：無法精確計算各時段價格與收盤價的偏差
2. **掛單深度**：無法評估各時段的流動性
3. **實際滑價**：無法模擬真實交易的市場衝擊

### 粗略替代方案

利用現有的 High/Low 資料：

- `(High - Low) / Close`：日內波動範圍，可用於估計執行偏差的上界
- `(Open - Close) / Close`：日內方向性偏差（已分析，-0.089%）
- `(High + Low) / 2 vs Close`：中位價與收盤價的偏差

### 若要精確分析

需新增資料源：
- **台股盤中行情**：5 分鐘或 1 分鐘 K 線（可從券商 API 或歷史行情供應商取得）
- **Tick 資料**：成交明細，可計算真實 VWAP 和各時段的流動性分佈

## 6. 結論與下一步

### 核心發現

1. **Label-執行 mismatch 是真實問題**：close-to-close 與 open-to-open 在 151 週差距 25 個百分點
2. **Qlib 用收盤價有充分學術理由**：資訊完整、噪音低、機構行為一致、模型穩定
3. **台股存在系統性「開高走低」**：日均 -0.089%，年化 ~22% 的隱性成本
4. **最佳執行窗口為 12:30-13:20**：波動率已低、流動性好、價格接近收盤
